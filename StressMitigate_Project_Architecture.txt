 1. PROJECT OVERVIEW
================================================================================

StressMitigate is an AI-powered mental health companion web application that
detects emotional stress through three modalities:
  • TEXT  — Analyzes typed messages using a fine-tuned BERT model
  • VOICE — Analyzes vocal tone using a Wav2Vec2-based TFLite model
  • FACE  — Analyzes facial expressions using a fine-tuned MobileNetV2 model

The system fuses these signals, sends them as context to Google's Gemini API,
and generates empathetic, personalized responses.

 2. TECHNOLOGY STACK
================================================================================

 Backend:
  • Python 3.10+
  • FastAPI (async web framework)
  • Motor (async MongoDB driver for Python)
  • Pydantic v2 (data validation & serialization)
  • bcrypt (password hashing)

  • MongoDB (document database, accessed via Motor async driver)

 Frontend:
  • Vanilla HTML5, CSS3, JavaScript (Single Page Application)
  • Manrope font (Google Fonts)
  • Material Icons Round (Google Icons)
  • No framework (no React/Vue/Angular — pure JS)

 Infrastructure:
  • Uvicorn (ASGI server)
  • .env file for secrets (GEMINI_API_KEY, MONGO_URI, JWT_SECRET)

── backend/ml_service.py ──────────────────────────────────────────────────────
  Role: Singleton ML model manager. Loads models at startup, provides inference.
  
  Models:
    1. Face Model (MobileNetV2):
       - Rebuilds architecture: MobileNetV2 → GAP → BN → Dense(128) → Dense(3)
       - Loads weights from .keras file
       - Also loads Haar Cascade XML for face pre-detection
       - predict_face(image_bytes):
         * Decodes image from bytes
         * Detects face using Haar Cascade (min 60x60)
         * Crops largest face
         * Converts BGR→RGB, resizes to 64x64, normalizes [0,1]
         * Returns: {"label": "High/Low/No Stress", "confidence": float}

    2. Voice Model (Wav2Vec2 TFLite):
       - Loads .tflite model into TF Lite interpreter
       - predict_voice(audio_bytes):
         * Loads audio via librosa at 16kHz
         * Detects silent audio (amplitude < 1e-4)
         * Pads/truncates to 80000 samples
         * Normalizes (zero-mean, unit-std)
         * Runs TFLite inference
         * Returns: {"label": "No/Low/High Stress", "confidence": float}

    3. Text Model (BERT):
       - Loads BertTokenizer + TFBertForSequenceClassification from local directory
       - predict_text(text):
         * Tokenizes with max_length=128
         * Runs BERT inference, applies softmax
         * Returns: {"label": "No Stress" or "STRESS DETECTED", "confidence": float}
  
  Edge Cases Handled:
    • Model file not found → Returns {"label": "unavailable"}
    • Import error (missing deps) → Same graceful fallback
    • Silent audio → Returns {"label": "silent_audio"}
    • No face detected → Returns {"label": "no_face_detected"}

── backend/brain.py ───────────────────────────────────────────────────────────
  Role: The "brain" — emotion fusion + Gemini API response generation.
  
  How it works (CRITICAL to understand):
  
    Step 1: _detect_conflict(emotions)
      - Checks if different modalities give contradictory signals
      - Example: Text says "No Stress" but Voice says "High Stress"
      - If conflict found, generates a warning for Gemini's system prompt
    
    Step 2: generate_response(emotions, user_text, communication_style)
      - Builds a dynamic SYSTEM PROMPT for Gemini containing:
        * AI persona ("You are Elysia, a compassionate AI mental-health companion")
        * Communication style from user preferences
        * Guidelines (keep responses 2-4 sentences, never diagnose)
        * Emotional context from each active modality with confidence %
        * Any conflict warnings
      - Sends the user's text as the USER MESSAGE
      - Uses google.generativeai SDK (genai) to call Gemini
      - On any error: returns LLM_FALLBACK constant
    
    Example System Prompt Sent to Gemini:
      "You are Elysia, a compassionate AI mental-health companion.
       Communicate in a **Gentle & Reassuring** tone.
       Keep responses concise (2-4 sentences), supportive, and safe.
       Never diagnose, prescribe, or replace professional help.
       Emotional context from user's multimodal inputs:
       Text analysis indicates **STRESS DETECTED** (confidence 94%).
       Voice tonality (Wav2Vec2) indicates **High Stress** (confidence 87%)."
    
    Then user's actual text is sent as the message.
    Gemini responds empathetically based on this hidden context.

================================================================================

 ┌──────────────────────────────────────────────────────────────┐

 └──────────────────────────────────────────────────────────────┘

 8. AI PIPELINE (HOW AI RESPONSES WORK)
================================================================================

 When a user sends a message in the AI Session:

 STEP 1: LOCAL ML ANALYSIS
   User's text → BERT model → "STRESS DETECTED" (confidence: 94%)
   User's audio → Wav2Vec2 TFLite → "High Stress" (confidence: 87%)
   User's image → MobileNetV2 → "Low Stress" (confidence: 62%)

 STEP 2: EMOTION FUSION (brain.py)
   Collects all modality results into a dict:
   {
     "text": {"label": "STRESS DETECTED", "confidence": 0.94},
     "voice": {"label": "High Stress", "confidence": 0.87},
     "face": {"label": "Low Stress", "confidence": 0.62}
   }

 STEP 3: CONFLICT DETECTION
   Checks if positive modalities (No Stress, Low Stress, Calm, Neutral)
   conflict with negative modalities (STRESS DETECTED, High Stress).
   In this example: Face (positive) conflicts with Text+Voice (negative).
   → Adds: "⚠️ CONFLICT DETECTED: face signals appear positive, but
     text, voice show distress. Gently check in on this discrepancy."

 STEP 4: SYSTEM PROMPT CONSTRUCTION
   Builds a hidden prompt for Gemini:
   ──────────────────────────────────────────────
   "You are Elysia, a compassionate AI mental-health companion.
   Communicate in a **Gentle & Reassuring** tone.
   Keep responses concise (2-4 sentences), supportive, and safe.
   Never diagnose, prescribe, or replace professional help.
   Emotional context from user's multimodal inputs:
   Text analysis indicates **STRESS DETECTED** (confidence 94%).
   Voice tonality (Wav2Vec2) indicates **High Stress** (confidence 87%).
   Facial expression (MobileNetV2) indicates **Low Stress** (confidence 62%).
   ⚠️ CONFLICT DETECTED: face signals appear positive, but text, voice
   show distress. Gently check in on this discrepancy."
   ──────────────────────────────────────────────

 STEP 5: GEMINI API CALL
   • Model: gemini-2.0-flash
   • System Instruction: [The prompt above]
   • User Message: [The user's actual typed text]
   • Gemini generates an empathetic response based on BOTH the hidden
     emotional context AND the user's actual words.

 STEP 6: RESPONSE DELIVERY
   Gemini's response is returned to the frontend and displayed in chat.
   The detected emotions are shown in the Emotion Insight sidebar.

 FALLBACK:
   If Gemini API fails (timeout, invalid key, network error):
   → Returns: "I'm here for you. Let's take a deep breath together.
     My connection is a bit slow right now, but I am listening."

     <div class="score-ring" style="--score: 72">
       <span class="score-value">72</span>

   ─────────────────────────────────────────────────────────

      Calm=100, Neutral=75, Anxious=40, Stressed=20
      Take average of today's check-in moods.

   b) AI session stress scores (40% weight):
      From BERT text analysis:
      "No Stress" → 100 points
      "STRESS DETECTED" conf < 70% → 60 points
      "STRESS DETECTED" conf 70-90% → 35 points

   c) Engagement bonus (20% weight):
      Did user check in today? +50 points
      Did user have an AI session? +50 points
      Full engagement = 100 points

   Daily Wellness Score = (a × 0.4) + (b × 0.4) + (c × 0.2)

   7-Day Rolling Average: Average of last 7 daily scores.
   New users: Show "--" (no data yet).

 10. AUTHENTICATION FLOW
================================================================================

 REGISTRATION:
   1. User fills email, display_name, password
   2. POST /api/v1/auth/register
   3. Backend checks if email exists in MongoDB
   4. Hashes password with bcrypt + random salt
   5. Saves user document to MongoDB
   6. Creates JWT (sub=user_id, email, exp=24h)
   7. Returns token + user_id + display_name
   8. Frontend stores token in localStorage

 LOGIN:
   1. User fills email, password
   2. POST /api/v1/auth/login
   3. Backend finds user by email (EXACT match — bug!)
   4. Verifies password with bcrypt
   5. Creates JWT, returns to frontend

 PROTECTED ENDPOINTS:
   All endpoints (except /auth/register and /auth/login) require:
   Authorization: Bearer <JWT>
   → Decoded by get_current_user() → Fetches user from MongoDB

   • sessionCount stored in localStorage('sm_sessions')
   • mediaRecorder for voice recording

 12. KNOWN ISSUES & BUGS
================================================================================

 BUG 1: Email is case-sensitive
   auth_router.py line 23: db.users.find_one({"email": req.email})
   Does not normalize to lowercase. "User@Email.com" ≠ "user@email.com"

 BUG 2: No password visibility toggle
   Login and register forms have type="password" with no eye button.

 BUG 3: Password not cleared on failed login
   The password field retains its value after a failed attempt.

 BUG 4: Wellness score is hardcoded (72)
   No calculation logic exists. Always shows 72 for all users.

 BUG 5: Dashboard mood buttons do nothing visible
   selectDashMood() calls submitQuickMood() which silently POSTs to API
   but provides zero visual feedback to the user.

 BUG 6: Sleep Sound button has no handler
   HTML button has no onclick attribute (lines 160-163 of index.html).

 BUG 7: Check-in has no success animation or redirect
   After submission, shows a text response but stays on the check-in page.

 BUG 8: Check-in does NOT analyze text for stress
   The optional_notes text is saved to MongoDB but never run through
   the BERT model. No stress_score or stress_category is saved.

 BUG 9: Gemini returns fallback instead of genuine response
   If GEMINI_API_KEY is invalid or the genai.configure() fails,
   every response will be the static fallback message.

 BUG 10: "Low Stress" never displayed for text analysis
   The BERT model only has 2 classes: "No Stress" and "STRESS DETECTED".
   There's no logic to produce "Low Stress" as an intermediate level.

 BUG 11: Voice recording doesn't confirm to user
   After recording, audioChunks are stored silently. No visual indicator
   of recording duration or that recording was captured.

 BUG 12: Camera capture is fire-and-forget
   Camera briefly flashes green on capture but no preview shown.

 BUG 13: Privacy toggles are UI-only
   togglePrivacy() only toggles a CSS class. The actual privacy_settings
   sent to the chat API always use hardcoded values.

 BUG 14: No conversation context (memory)
   Each message to Gemini is standalone. No previous messages are sent.
   This means the AI has no memory of the conversation.

 BUG 15: Settings page items are unimplemented
   "Personal Information", "Security", "Privacy Settings", "Help Center"
   all have no click handlers — they show chevron_right but do nothing.

 BUG 16: No Exercise tab in navigation
   Only Dashboard, Check-In, AI Session, and Settings exist.

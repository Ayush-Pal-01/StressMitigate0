 1. EXECUTIVE SUMMARY
================================================================================

StressMitigate is an AI-powered mental health companion that detects emotional
stress through three modalities (Text/BERT, Voice/Wav2Vec2, Face/MobileNetV2),
fuses them, and generates empathetic responses via Google Gemini.

The application has grown beyond the prototype stage. It currently has:
  ✅ Working auth (register/login with JWT + bcrypt, email normalized)
  ✅ Multimodal chat (text, voice, face — all 8 hybrid cases handled)
  ✅ Conversation memory (brain.py uses start_chat with last 5 messages)
  ✅ 3-tier text stress classification (No/Low/High threshold logic)
  ✅ Wellness score engine (daily score + 7-day rolling average)
  ✅ Weekly report generation, recommendations, stress trigger analysis
  ✅ Breathing exercise widget
  ✅ Settings modals (profile edit, password change, privacy sync)
  ✅ Gemini retry with model fallback (gemini-2.5-flash → gemini-3.1-flash-lite)
  ✅ Check-in BERT analysis (checkin_router runs predict_text on notes)
  ✅ Chart.js wellness trend chart (30-day history)

This PRD defines what remains to transform it into a **production-grade,
professionally deployable** application.

 3. VERIFIED & APPROVED UPGRADES
================================================================================

── A. MACHINE LEARNING & INFERENCE OPTIMIZATION ──
VERDICT: APPROVED & CRITICAL
Priority: HIGH

  1. BERT → ONNX + INT8 Quantization
     - Current: tf_model.h5 at 438MB, loaded via TFBertForSequenceClassification
     - Target: ONNX INT8 model at ~100MB, loaded via onnxruntime
     - Why: 4x size reduction, 2-3x inference speedup on CPU
     - Script needed: convert_bert_onnx.py (export TF → ONNX → quantize INT8)
     - ml_service.py: Add new predict_text_onnx() method alongside existing

  2. MobileNetV2 (.keras) → TFLite
     - Current: fine_tuned_stress_model.keras at ~26MB via full TensorFlow
     - Target: TFLite model at ~10MB via tf.lite.Interpreter
     - Why: Unifies inference pipeline (Voice already uses TFLite),
       removes need to load full TensorFlow for face model
     - Script needed: convert_face_tflite.py

  3. Decoupled Inference (Non-Blocking)
     - PHASE 1 (immediate): Wrap all predict_*() calls in asyncio.to_thread()
       in chat_router.py and checkin_router.py
     - PHASE 3 (infra): Move to Celery + Redis worker queue for full decoupling

  4. Fix Fake Fallback Predictions
     - predict_text() line 121: returns hardcoded STRESS DETECTED when model
       missing — MUST return {"label": "unavailable"} instead
     - predict_face() line 145: returns hardcoded "Stressed" — same fix
     - predict_voice() line 185: returns hardcoded "High Stress" — same fix

── E. SECURITY & DATA COMPLIANCE ──
VERDICT: APPROVED & MANDATORY
Priority: HIGH

  1. File Upload Validation
     - Audio: Only accept audio/webm, audio/wav, audio/mp3 — max 10MB
     - Image: Only accept image/jpeg, image/png — max 5MB
     - Reject everything else with 415 Unsupported Media Type

  2. MongoDB Encryption at Rest
     - Enable MongoDB Enterprise encryption or use mongocryptd
     - Encrypt conversations and checkins collections
     - Index on user_id remains unencrypted for query performance

  3. Media Purge Policy
     - Audio/image bytes MUST be purged from memory immediately after inference
     - NEVER persist raw audio/video to disk or database
     - Only store emotion labels and confidence scores

  4. JWT Hardening
     - Move JWT_SECRET to a proper secret manager (or at minimum, a strong
       randomly generated key — current one is a readable string)
     - Consider refresh token rotation for sessions > 24h

  5. Input Sanitization
     - Sanitize user text before sending to Gemini (prevent prompt injection)
     - Sanitize display_name and optional_notes for XSS
